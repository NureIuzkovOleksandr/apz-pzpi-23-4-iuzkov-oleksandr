3 ОПИС ВИКОНАНОЇ РОБОТИ
     3.1 Загальні відомості про систему
     Назва системи: Visual Studio Code (VS Code).
     Розробник: Microsoft Corporation.
     Рік першого випуску: 2015.

     Visual Studio Code — це безкоштовний редактор вихідного коду з відкритим вихідним кодом, розроблений компанією Microsoft. Система поширюється за ліцензією MIT та є одним із найпопулярніших інструментів розробки у світі: за даними щорічного опитування Stack Overflow Developer Survey, VS Code займає першу позицію серед середовищ розробки вже протягом кількох років поспіль.
     Редактор підтримує понад 30 мов програмування «з коробки» та розширюється за допомогою системи розширень (extensions). VS Code орієнтований на такі категорії користувачів: веброзробники, розробники програмного забезпечення, дата-сайентисти, DevOps-інженери. Ключові функції системи включають підсвічування синтаксису, автодоповнення коду (IntelliSense), вбудований відладчик, інтеграцію з Git, термінал, підтримку розширень.
     
     3.2 Архітектурний стиль системи
     Visual Studio Code побудований на основі багатошарової модульної архітектури з елементами мікроядра (Microkernel / Plugin Architecture). Система реалізована на платформі Electron, яка поєднує рушій Chromium для відображення інтерфейсу та середовище виконання Node.js для серверної логіки.
     
     
     
     Архітектурний стиль VS Code можна охарактеризувати як гібридний:
* модульна архітектура — ядро редактора надає мінімальний набір базових можливостей, а вся розширена функціональність реалізується через незалежні модулі;
* архітектура на основі подій (Event-Driven Architecture) — компоненти взаємодіють через систему подій та повідомлень, що забезпечує слабку зв'язаність;
* клієнт-серверна архітектура (Language Server Protocol) — мовна підтримка реалізована через окремі мовні сервери, що взаємодіють з редактором за протоколом LSP.

     Вибір платформи Electron обґрунтований тим, що вона дозволяє розробляти крос-платформні настільні застосунки з використанням веб-технологій (HTML, CSS, TypeScript), забезпечуючи нативний вигляд та продуктивність на операційних системах Windows, macOS і Linux.
     
     3.3 Основні компоненти системи
     Архітектура VS Code складається з таких основних підсистем.
     Electron Shell — оболонка застосунку. Відповідає за запуск процесів, управління вікнами, системну інтеграцію (меню, трей, нативні діалоги). Поділяється на головний процес (main process) та процеси рендерингу (renderer processes).
     Workbench — основний інтерфейс користувача. Реалізований на TypeScript та HTML/CSS. Включає Activity Bar, Side Bar, Editor Area, Panel та Status Bar. Workbench є центральним компонентом, що координує відображення всіх UI-елементів.
     Editor Core (Monaco Editor) — компонент редагування тексту. Monaco — це незалежна бібліотека, що реалізує функції редактора: підсвічування синтаксису, множинний курсор, згортання коду, diff-перегляд. Monaco також використовується у вебзастосунках незалежно від VS Code.
     Extension Host — ізольований процес для виконання розширень. Розширення запускаються в окремому процесі Node.js, що унеможливлює блокування основного інтерфейсу при помилках у розширеннях. Взаємодія з Workbench відбувається через строго визначений API.
     Language Server Protocol (LSP) — протокол взаємодії з мовними серверами. Кожна мова програмування може мати власний мовний сервер, що надає функції автодоповнення, перевірки помилок, переходу до визначення тощо. LSP стандартизує протокол взаємодії між редактором і мовним сервером.
     Debug Adapter Protocol (DAP) — протокол взаємодії з відладчиками. Аналогічно до LSP, DAP стандартизує взаємодію між відладчиком та редактором.
     File System & Storage — підсистема роботи з файлами та збереження налаштувань. Забезпечує доступ до файлової системи, управління воркспейсом, збереження користувацьких налаштувань та стану редактора.
     
     3.4 Взаємодія компонентів
     VS Code використовує модель багатопроцесорної архітектури. Виділяються такі основні процеси та канали їх взаємодії.
     Головний процес (Main Process) — запускається першим, управляє життєвим циклом застосунку та дочірніми процесами. Взаємодіє з операційною системою для відображення нативних вікон.
     Процес рендерингу (Renderer Process) — відповідає за відображення інтерфейсу Workbench. Запущений у контексті Chromium, має доступ до DOM та Web API.
     Процес Extension Host — ізольований процес Node.js для виконання розширень. Взаємодіє з процесом рендерингу через IPC (Inter-Process Communication) з використанням JSON RPC.
     Мовні сервери (Language Servers) — зовнішні процеси, що запускаються Extension Host. Взаємодіють з редактором через Language Server Protocol поверх stdio або сокетів.
     Потік взаємодії при автодоповненні коду: користувач вводить символ → Monaco Editor генерує запит → Extension Host отримує подію → Мовний сервер обчислює варіанти → відповідь повертається до Monaco Editor → відображається список підказок. Весь цей процес є асинхронним і не блокує UI.
     3.5 Масштабованість та надійність
     VS Code забезпечує масштабованість та надійність завдяки таким механізмам.
     Ізоляція розширень. Кожне розширення виконується в Extension Host — окремому процесі Node.js. Якщо розширення падає або зависає, це не впливає на основний UI редактора. Користувач отримує повідомлення про проблему і може вимкнути проблемне розширення.
     Ліниве завантаження (Lazy Loading). Розширення активуються не при запуску, а за певними подіями (відкриття файлу певного типу, виконання команди). Це суттєво прискорює час запуску редактора.
     Language Server Protocol. LSP дозволяє мовним серверам працювати як незалежні процеси. Якщо мовний сервер падає — його можна перезапустити без перезавантаження редактора. Один мовний сервер може обслуговувати кілька відкритих файлів.
     Remote Development. VS Code підтримує архітектуру з розділеним середовищем: UI запускається локально, а весь код виконується на віддаленій машині (SSH, контейнер Docker, WSL). Це реалізовано через розширення Remote Development та спеціальний VS Code Server.
     Web-версія (vscode.dev). VS Code може запускатися повністю у браузері. Monaco Editor та Workbench є веб-сумісними, що дозволяє редагувати файли з GitHub або Azure Repos без встановлення застосунку.


























4 ВИСНОВКИ
     У результаті виконання практичного заняття було:
* досліджено архітектуру програмної системи Visual Studio Code;
* визначено архітектурний стиль — гібридна модульна архітектура на базі Electron із використанням мікроядра та подієво-орієнтованої взаємодії;
* проаналізовано основні компоненти: Workbench, Monaco Editor, Extension Host, LSP, DAP;
* описано взаємодію компонентів через IPC, JSON RPC та стандартизовані протоколи LSP і DAP;
* оцінено механізми надійності та масштабованості: ізоляція розширень, ліниве завантаження, Remote Development, веб-версія;
* сформовано практичні навички архітектурного аналізу програмних систем.

     Архітектура Visual Studio Code є прикладом добре спроєктованої розширюваної системи. Ключовим архітектурним рішенням є винесення мовної підтримки та розширень у окремі ізольовані процеси, що забезпечує стабільність ядра редактора незалежно від якості сторонніх розширень. Використання стандартизованих протоколів LSP та DAP дозволило сформувати ціле екосистему інструментів, сумісних з різними редакторами.
