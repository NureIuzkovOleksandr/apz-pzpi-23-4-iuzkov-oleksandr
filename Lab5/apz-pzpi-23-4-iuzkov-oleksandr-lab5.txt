     Метою лабораторної роботи є підготовка та представлення фінальної презентації цілісного проекту «SmartClimate», розробленого протягом двох семестрів навчання.
     Відповідно до завдання необхідно:
– Підготувати коротку презентацію цілісного проекту (слайди), що описує архітектуру, технологічний стек, цілі та рамки системи.
– Проаналізувати та задокументувати заплановані на початку розробки функції порівняно з фактично виконаними у MVP-релізі (визначити, що встигли зробити, а що — ні).
– Підготувати демонстраційне відео (тривалістю 4–6 хвилин) основних функцій проекту (записати заздалегідь) у форматі презентації або elevator pitch продукту.
– Оформити матеріали презентації та звіт відповідно до встановлених вимог.
     
     
     
     
     
     
     
     
     
     
     
     
     

3 ОПИС ВИКОНАНОЇ РОБОТИ
     3.1 Загальні відомості про проект
     Проект SmartClimate — це інтегрована IoT-система моніторингу та автоматизованого управління мікрокліматом (температурою та вологістю) у приміщенні. Система призначена для підвищення комфорту користувачів та оптимізації витрат енергії шляхом автоматичного керування кліматичною технікою (кондиціонер, опалення, зволожувачі, осушувачі) на основі заданих порогових значень та аналітики телеметрії.
     3.2 Архітектура та технологічний стек
     Система розроблена за багаторівневою клієнт-серверною архітектурою:
– Апаратний рівень (IoT): Мікроконтролер ESP32 (програма на C++/Arduino), цифровий датчик DHT22 для збору показників навколишнього середовища, світлодіод для локальних аномалій та сервопривід для симуляції роботи механічних актуаторів техніки. Зв'язок відбувається через протокол MQTT (брокер Eclipse Mosquitto).
– Серверний рівень (Backend): Реалізований на Python 3 з використанням асинхронного фреймворку FastAPI. База даних — PostgreSQL (ORM SQLAlchemy). Забезпечує збір телеметрії, автентифікацію через JWT-токени, логування подій та аномалій, а також автоматичне прийняття рішень для керування пристроями (AutoControlFlow).
– Клієнтський рівень (Web & Mobile): Web Dashboard реалізовано на React + Vite + TS для відображення інтерактивної температурної карти та налаштувань. Мобільний додаток реалізовано на Flutter (Dart) для кросплатформної підтримки (Android / iOS) з функцією віддаленого контролю та push-сповіщень.
– Інфраструктурний рівень (DevOps): Повна контейнеризація в Docker, розгортання в локальному кластері Kubernetes (Minikube) із сервісами скейлінгу подів за допомогою HPA та тестуванням навантаження через Locust.

4 ВИСНОВКИ
     У результаті виконання лабораторної роботи було успішно систематизовано та презентовано результати роботи над проектом «SmartClimate» за два семестри розробки. Створено звіт та веб-презентацію, які наочно демонструють архітектурну зрілість системи, її працездатність та готовність до масштабування.
     Проект успішно пройшов шлях від концептуальних вимог (Vision & Scope) та проектування бази даних до повної апаратної симуляції (ESP32), розробки стабільного REST API на FastAPI, побудови клієнтських інтерфейсів (React та Flutter) та розгортання стійкої, масштабованої мікросервісної хмарної інфраструктури в Kubernetes кластері.
